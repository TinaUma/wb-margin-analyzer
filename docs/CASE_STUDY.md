# WB Margin Analyzer — кейс для собеседования

## Боль которую решает

WB-продавец с 200+ SKU не видит реальную картину прибыльности. Платформа берёт комиссию 15–25% + логистику туда + логистику обратно по тарифу ×1.5 при возвратах. В итоге товар который выглядит прибыльным — уходит в минус. Существующие решения — это либо Excel вручную, либо дорогие аналитические сервисы с подпиской.

## Что сделано

Полноценный SaaS-инструмент с нуля: от ТЗ до деплоя на VPS.

**Пользователь загружает два Excel-файла** (закупки + продажи) → система рассчитывает маржу по каждому товару → выдаёт цветную таблицу с AI-диагнозом и конкретными рекомендациями.

## Ключевые решения

- **Асинхронная обработка**: загрузка файла возвращает `202 Accepted` мгновенно, анализ идёт в фоне — фронтенд поллит статус каждые 2 сек
- **What If симулятор**: слайдеры цены/себестоимости пересчитывают маржу в реальном времени без запросов к API
- **AI-интерпретация**: Claude получает данные по всем 200 товарам и генерирует структурированный отчёт: Диагноз / Рекомендации / Риски
- **Excel-экспорт с авторизацией**: файл скачивается через Axios blob (не через `<a href>`) чтобы передать JWT-токен
- **Цветная таблица**: зелёная ≥25% / жёлтая 10–25% / красная <10% — с фильтрами по зонам, сортировкой и модалкой карточки товара

## Что нетривиально технически

- `docker compose restart` не перечитывает `.env` — нужен `--force-recreate` (поймала в проде)
- Anthropic API блокирует RU-серверы — решила через info-баннер + скриншоты в README вместо скрытия фичи
- Формула возврата: `логистика_возврат = базовая_ставка × 1.5 × кол-во_возвратов` — специфика WB-тарифа
